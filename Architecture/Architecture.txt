Описание многопользовательской клиент-серверной системы:

Интернет-магазин корейской косметики. Посетитель заходит на сайт, ищет товар, отправляет в корзину нужные товары, оплачивает заказ.

Тип архитектуры:
Трехуровневая клиент-серверная.

Данный тип архитектуры позволяет получать запрашиваемые данные, например, тканевые маски MJ CARE.
Данные о товарах и их наличии хранятся в БД.
Преимущества: Меньшая нагрузка на клиентское приложение, между клиентской программой и сервером приложения передается лишь
минимально необходимый поток данных, сервер приложения может быть запущен в одном или нескольких экземплярах на одном или
нескольких компьютерах.

Производительность: 

Выбранная архитектура позволяет использовать высокоскоростные сетевые соединения между серверами и отдавать на клиент только результат запроса.

Безопасность:

Клиент не имеет прямого доступа к серверу БД.

Надежность:

Сервер приложений может кэшировать ответы от сервера БД и использовать их на время недоступности сервера БД.

Компоненты:

1. Слой клиента. Главная функция - представление задач и результатов, понятных пользователю.
Представлен тонким клиентом. Необходим для запуска сетевого приложения. Преимущество - меньшая нагрузка на клиентское приложение.
2. Слой логики. Он обрабатывает команды, выполняет логические решения и вычисления.
Связывает, обрабатывает и перемещает данные между окружающими слоями. Сервер приложений.
3. Взаимодействие с иными ресурсами. Связан с сервером приложений и необходим для заказа товара. 
4. Слой данных. Здесь хранится вся необходимая информация. Сервер БД.

Клиент делает запрос, который обрабатывается в сервере приложений и отправляется в сервер БД.
Затем, информация из БД обрабатывается и отправляется клиенту.
